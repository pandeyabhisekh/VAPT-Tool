Missing Rate-Limiting on Login Endpoint

Medium
Description

The application does not enforce rate limiting on the login endpoint, allowing rapid automated attempts.

Impact Assessment

Attackers can perform high-speed brute-force and credential stuffing attacks to compromise user accounts.

Steps to Reproduce

1. Send 5 POST requests to http://localhost:8081/ in < 1 second. 2. Observe no blocking or delays.

Proof of Concept / Technical Evidence
5 requests completed in 0.47 seconds.
Recommendation & Fix

Remediation Strategy: Implement rate limiting (e.g., 5 requests per minute per IP) and add CAPTCHA after 3 failed attempts.

Technical Fix Details:

Nginx: limit_req_zone $binary_remote_addr zone=login:10m rate=5r/m;
Application: Use libraries like 'Flask-Limiter'.

Missing Security Header: Permissions-Policy

                Low
            

            

                

                    
Description

                    
The security-enhancing header 'Permissions-Policy' is not present in the HTTP response.

                


                
                

                    
Impact Assessment

                    
Attackers can exploit the lack of browser-side security controls to perform XSS, clickjacking, or data leakage.

                

                

                
                

                    
Steps to Reproduce

                    
1. Send a GET request to http://localhost:8081.
2. Inspect headers in the response.

                

                

                

                    
Proof of Concept / Technical Evidence

                    
Header 'Permissions-Policy' missing from response at http://localhost:8081

                


                

                    
Recommendation & Fix

                    
Remediation Strategy: Define a Permissions-Policy to restrict browser features like camera or geolocation.

                    
                    
Technical Fix Details:

                    
In Nginx: add_header Permissions-Policy 'value';
In Flask: response.headers['Permissions-Policy'] = 'value'

                    
                

            

        

        
        

            

                
Exposed Administrative Panel

                Medium
            

            

                

                    
Description

                    
An administrative login or dashboard was discovered at a common path.

                


                
                

                    
Impact Assessment

                    
Provides a direct target for brute-force attacks and unauthorized administrative access if weak credentials are used.

                

                

                
                

                    
Steps to Reproduce

                    
1. Navigate to http://localhost:8081/admin.
2. Observe the administrative login interface.

                

                

                

                    
Proof of Concept / Technical Evidence

                    
Found login form at http://localhost:8081/admin

                


                

                    
Recommendation & Fix

                    
Remediation Strategy: Restrict access to the admin panel using IP whitelisting or a VPN. Change the default /admin path to a non-obvious URL.

                    
                    
Technical Fix Details:

                    
Nginx: allow 192.168.1.0/24; deny all; inside the admin location block.

Exposed Git Repository

                High
            

            

                

                    
Description

                    
The .git directory is accessible, allowing attackers to download the entire source code history and internal configurations.

                


                
                

                    
Impact Assessment

                    
Source code theft, exposure of hardcoded secrets, and full reverse engineering of the application logic.

                

                

                
                

                    
Steps to Reproduce

                    
1. Access http://localhost:8081/.git/config or http://localhost:8081/.git/index in a browser.
2. Use 'git-dumper' to pull the full repository.

                

                

                

                    
Proof of Concept / Technical Evidence

                    
GET http://localhost:8081/.git returned HTTP 302

                


                

                    
Recommendation & Fix

                    
Remediation Strategy: Block access to the .git directory immediately and clean any leaked secrets from the repository history before redeploying.

                    
                    
Technical Fix Details:

                    
Nginx: location ~ /\.git { deny all; }
Apache:  Order allow,deny Deny from all 

                    
                

            

        

        
        

            

                
Exposed Environment File (.env)

                Critical
            

            

                

                    
Description

                    
The .env file containing sensitive environment variables like DB credentials and API keys is publicly accessible.

                


                
                

                    
Impact Assessment

                    
Immediate full system compromise. Attackers can access databases, third-party services, and cloud infrastructure.

                

                

                
                

                    
Steps to Reproduce

                    
1. Send a GET request to http://localhost:8081/.env.
2. Read sensitive keys like DB_PASSWORD or AWS_SECRET.

                

                

                

                    
Proof of Concept / Technical Evidence

                    
Successfully accessed http://localhost:8081/.env with content length 166

                


                

                    
Recommendation & Fix

                    
Remediation Strategy: Move the .env file outside of the web root and rotate all secrets contained within it immediately.

                    
                    
Technical Fix Details:

                    
Move .env to a parent directory not served by the web server. Use environment variables injected by the OS or Docker instead of a file.
